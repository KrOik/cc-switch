## 🎯 完善计划（按优先级排序）

### 阶段 1：核心表单输入组件（基础设施）

**目标**：实现通用的表单输入组件，为所有添加/编辑功能提供基础。

#### 1.1 文本输入组件
- **功能**：单行文本输入框
- **特性**：
  - 光标显示和移动
  - 文本编辑（插入、删除、退格）
  - 复制/粘贴支持（可选）
  - 输入验证（必填、格式校验）
- **使用场景**：Provider 名称、URL、API Key 等

#### 1.2 多行文本编辑器
- **功能**：多行文本编辑
- **特性**：
  - 滚动支持
  - 行号显示（可选）
  - 语法高亮（JSON/TOML，可选）
- **使用场景**：配置文件编辑、MCP 命令参数

#### 1.3 选择器组件
- **功能**：下拉选择、单选、多选
- **特性**：
  - 键盘导航
  - 搜索过滤（可选）
- **使用场景**：Provider 类型选择、应用选择

#### 1.4 表单容器
- **功能**：管理多个输入字段
- **特性**：
  - 字段间导航（Tab/Shift+Tab）
  - 表单验证
  - 提交/取消处理
- **使用场景**：所有添加/编辑界面

**预估工作量**：中等（需要仔细设计交互逻辑）

---

### 阶段 2：Provider 完整功能

**目标**：实现 Provider 的完整 CRUD 操作。

#### 2.1 Provider 添加界面
- **表单字段**：
  - Provider ID（自动生成或手动输入）
  - 名称（必填）
  - 类型/分类（下拉选择）
  - API Key（密码输入）
  - Base URL（URL 验证）
  - 网站 URL（可选）
  - 备注（多行文本）
  - 图标和颜色（可选）
- **操作流程**：
  1. 按 `A` 键打开添加界面
  2. 填写表单字段
  3. Tab 键切换字段
  4. Enter 提交，Esc 取消
  5. 验证输入并保存到数据库
  6. 刷新 Provider 列表

#### 2.2 Provider 编辑界面
- **功能**：编辑现有 Provider
- **操作流程**：
  1. 选中 Provider 后按 `E` 键
  2. 加载现有数据到表单
  3. 修改字段
  4. 保存更新

#### 2.3 Provider 切换逻辑完善
- **当前状态**：UI 完成，业务逻辑标记为 TODO
- **需要实现**：
  1. 更新数据库中的 current provider
  2. 写入对应应用的 live 配置文件
  3. 刷新缓存
  4. 显示成功/失败消息

#### 2.4 Provider 删除逻辑完善
- **当前状态**：确认对话框完成
- **需要实现**：
  1. 从数据库删除
  2. 从 live 配置移除（如果是当前 provider）
  3. 刷新列表

**预估工作量**：较大（涉及表单、验证、配置文件操作）

### 阶段 3：MCP 完整功能

**目标**：实现 MCP 服务器的完整管理功能。

#### 3.1 MCP 添加界面
- **表单字段**：
  - 服务器 ID（必填）
  - 名称（可选）
  - 描述（可选）
  - 命令类型（stdio/sse）
  - 命令（必填）
  - 参数（数组输入）
  - 环境变量（键值对输入）
  - 启用的应用（多选：Claude/Codex/Gemini/OpenCode/OpenClaw）
- **操作流程**：
  1. 按 `A` 键打开添加界面
  2. 填写表单
  3. 选择启用的应用
  4. 保存并同步到各应用配置

#### 3.2 MCP 编辑界面
- **功能**：编辑现有 MCP 服务器
- **操作流程**：类似添加界面

#### 3.3 MCP 应用启用/禁用切换
- **功能**：快速切换 MCP 在各应用中的启用状态
- **操作方式**：
  - 选中 MCP 后按数字键（1-5）切换对应应用
  - 或进入详情界面使用 Space 键切换
- **需要实现**：
  1. 更新数据库
  2. 同步到对应应用的 live 配置
  3. 实时更新 UI 显示

**预估工作量**：中等
